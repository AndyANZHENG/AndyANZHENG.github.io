ES6 Standard

Learning objectives

  • Be able to read and write new ES6 standard

The language “JavaScript” follows certain standards, similar to how a spoken language has certain grammar. However, the JavaScript language has evolved a lot faster than a language such as English or French. While in the past the language evolved quite sporadically. This is set to change with updates coming more frequently.

One of the newest standards for JavaScript - ES6 - was released in 2015. Since it always takes time to implement the new standards into different frameworks and environments, it is only now, that we are starting to really see the new conventions become the industry standards. However older browsers which aren’t updated anymore will be unable to understand the new JavaScript languages, which is why there are still a lot of people who write in previous standards.

Functions

So far we have declared functions with function (). But in ES6 we use () =>

// ES5
function add(a, b) {
  return a + b;
}
// ES6 = defines the function add, taking two inputs (a and b) and adds them together
var add = (a, b) => {
  return a + b;
};
// shorten it even more
// notice how `return` is omitted
var add = (a, b) => a + b;

// shorten even more (slightly different code, we double one input here)
var double = a => a * 2;

As you can see above, we can remove the curly brackets { } if we only have one line of logic and if we don’t need a return statement either.

If we only have one parameter we don’t need to use the round brackets ( ) either. However if we have 0 or 2+ parameters we need to use the brackets.

You will notice that byusing ES6 we can write a lot less code - which some also call ‘clean code’. However, it can sometimes become too little, making it again difficult to understand.

Variables

So far we declared variables using the var statement. ES6 has two different variable declarations. For variables that won’t change we use const. For variables that will change throughout the code we use let. So let is just a replacement for var, whereas const is different in that it cannot be changed.

// old way
var price = 2;
var revenue = 0;

function revenue(sold) {
  revenue = price * sold;
}

// ES6
const PRICE = 2;
let revenue = 0;

const revenue = sold => {
  revenue = PRICE * sold;
};

Why should we use const at all? Imagine you have a lot of variable declarations, it becomes quickly very hard to tell what a variable is used for. Using const and let gives the reader the ability to better understand the purpose of a variable. Also using const prevents you from accidentally overwriting a variable.

Object literals

A JavaScript object literal is a comma-separated list of name-value pairs wrapped in curly braces. Object literals encapsulate data, enclosing it in a tidy package. This minimizes the use of global variables which can cause problems when combining code.

// ES5
function saveFile(file, data) {
  return {
    file: file,
    data: data,
    retrieve: function() {
      // do something
    }
  };
}

// ES6
function saveFile(file, data) {
  return {
    file,
    data,
    retrieve() {
      // do something
    }
  };
}

As you can see if property name and property variable are the same (file, data) we only have to write them once. Equally we can get rid of the word function when declaring the retrieve method.

Why and How We Use Object Literals

Several JavaScripts from dyn-web use object literals for setup purposes. Object literals enable us to write code that supports lots of features yet still provide a relatively straightforward process for implementers of our code. There is no need to invoke constructors directly or maintain the correct order of arguments passed to functions.

Object literals are also useful for unobtrusive event handling; they can hold the data that would otherwise be passed in function calls from HTML event handler attributes.

There is one drawback: if you are unfamiliar with the syntax, it can be very easy to cause errors which cause the code to stop working completely.

Destructuring

ES6 allows us to write even more concise code. If we want to pull some properties of objects and save them in variables, we can use destructuring.

Destructuring is the opposite of data construction. Rather than constructing a new object or array, for example, destructuring takes data from an existing object or array and literally destructs it to extract only the values you’re interested in.

It does this through the use of a pattern that’s used by ES6 to match the values that you want to extract. This pattern mirrors the structure of the data item you’re destructuring, with only those parts of the data item that match the pattern, being extracted.

function printInfo(firstName, lastName, age) {
  // ES5
  console.log(
    'Hi I am ' +
      firstName +
      ', my last name is ' +
      lastName +
      '. I am ' +
      age +
      ' years old!'
  );

  // ES6 way
  console.log(
    `Hi I am ${firstName}, my last name is ${lastName}. I am ${age} years old!`
  );
}

Instead of using quotations marks " " we use backticks . No need for + and worry about putting all the space after variables. instead wrap all JS variables or logic in ${ }. You can even put functions and complex logic in there.

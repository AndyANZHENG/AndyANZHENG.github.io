this keyword

As you might remember, we can define functions within objects (known as methods). When we can call methods they access the properties of the object that they belong to. To access these properties we use the this key to declare the object's property. The this keyword is very contextual and will usually refer to the object that it is associated with.

var dog = {
  nickname: 'Benji',
  call() {
    console.log('Come here ' + this.nickname);
    // by calling this.nickname here, we can access the nickname properties
    // of the object dog
  }
};
dog.call(); // logs 'Come here Benji'

Let’s take a step back and look closer at the this keyword…

var dog = {
  nickname: 'Benji',
  whatIsThis() {
    // we're inside the object declaration
    console.log(this);
  },
  call() {
    // also inside the object declaration
    console.log(this.nickname);
  }
};
dog.whatIsThis(); // { nickname: 'Benji', call: [Function: call] }
dog.call(); // 'Benji'

The value of this is the object {nickname:’Benji’, call: Function: call} hence why this.nickname logs Benji

When this refers to the global object

Using this we are able to access the property nickname from the same object.

If you use this outside of an object declaration it refers to the global object. Below are two examples:

// in Node:
var obj = {
  exampleProperty: 10,
  exampleMethod() {
    // here we're inside the object declaration
    function exampleFunction() {
      // we're outside of the object declaration
      console.log(this === global); // as a result 'this' refers to the global object
    }
    exampleFunction();
  }
};
obj.exampleFunction(); // true, this is equal to the global object

// in browser:
var obj = {
  exampleProperty: 10,
  exampleMethod() {
    // here we're inside the object declaration
    function exampleFunction() {
      // we're outside of the object declaration
      console.log(this === window); // as a result 'this' refers to the window object
    }
    exampleFunction();
  }
};
obj.exampleFunction(); // true , this is equal to the window object

The problem with the ‘this’ object and inner functions

Going back to the dog example, we can demonstrate how you can easily make a mistake while using this.

var dog = {
  nickname: 'Benji',
  call() {
    console.log('first call nickname is showing => ' + this.nickname);
    function secondCall() {
      console.log('second call nickname is undefined => ' + this.nickname);
    }
    secondCall();
  }
};

dog.call();
// logs "first call nickname is showing => Benji"
// logs "second call nickname is undefined => undefined"

In the first console.log, the nickname Benji is correctly showing. But in the second console.log that is run by the secondCall method, the nickname is undefined!

Why does this happen?

  • LINE 3: call() is a method of the dog object, so this will refer to the dog object
  • LINE 6: The call() method creates an inner function called secondCall() which has its own scope. (Scope is extremerly important while using the 'this' keyword.
  • Within the scope of the secondCall() method, this no longer refers to the dog object, in fact it refers to the global object.
  • The call() method did not pass the dog object along to the secondCall() function. This means secondCall() does not have access to the properties of the dog object!

How can we solve this?

Passing the this object to an inner function

var dog = {
  nickname: 'Benji',
  call() {
    var props = this;
    function secondCall() {
      console.log('Come here' + props.nickname);
    }
    secondCall();
  }
};
dog.call(); // logs 'Come here Benji'

The nickname Benji is showing properly! So what did we do here?

  • We store the this object in a local variable
  • The inner function secondCall() can “see” the props variable because props and secondCall() are declared within the call() function
  • So ‘Benji’ can be accessed using props.nickname

Remember: this is used inside a function (let’s say function A) and it contains the value of the object that invokes function A. We need 'this' to access methods and properties of the object that invokes function A, especially since we don’t always know the name of the invoking object, and sometimes there is no name to use to refer to the invoking object.
